# Requisitos No Funcionales — NN Auth System (Spring Boot Java)

> Proyecto: NN Auth System — Spring Boot Edition
> Tipo: Proyecto educativo — SENA
> Stack: Spring Boot 3 (Java 21) + React + PostgreSQL 17

---

# RNF-001 — Seguridad

**Tipo**: Requisito No Funcional de Seguridad
**Referencia**: OWASP Top 10 2021 — ver `_docs/conceptos/owasp-top-10.md`

## Requisitos Específicos

**RNF-001.1** — Las contraseñas deben hashearse con BCrypt (factor de costo mínimo 10)
antes de persistirse. Nunca almacenar contraseñas en texto plano.

**RNF-001.2** — Los tokens JWT deben firmarse con secretos de mínimo 32 caracteres
aleatorios. Si `JWT_SECRET` tiene menos de 32 chars, la aplicación debe rechazar el boot.

**RNF-001.3** — Los endpoints de autenticación deben tener rate limiting
(Bucket4j, máx. 10 req / 15 min por IP).

**RNF-001.4** — La API debe usar HTTPS en producción (TLS 1.2+).

**RNF-001.5** — Las cabeceras HTTP de seguridad deben configurarse en `SecurityConfig.java`
(CSP, HSTS, X-Frame-Options, X-Content-Type-Options).

**RNF-001.6** — El CORS debe configurar orígenes explícitos en `SecurityConfig.java` —
nunca `allowedOrigins("*")` en producción.

**RNF-001.7** — Toda consulta a la BD debe usar Spring Data JPA Repository o JPQL
(nunca SQL crudo con concatenación de strings).

**RNF-001.8** — Swagger UI (`/swagger-ui.html`) debe deshabilitarse cuando
`ENVIRONMENT=production`.

**RNF-001.9** — Los mensajes de error de autenticación deben ser genéricos —
no revelar si el email existe o si es la contraseña la incorrecta.

# RNF-002 — Rendimiento

**Tipo**: Requisito No Funcional de Rendimiento

## Requisitos Específicos

**RNF-002.1** — El endpoint de login debe responder en menos de 500 ms en condiciones
normales (incluyendo verificación BCrypt).

**RNF-002.2** — El pool de conexiones PostgreSQL (HikariCP, incluido en Spring Boot)
debe configurarse con mínimo 2 y máximo 10 conexiones.

**RNF-002.3** — El frontend debe cargar el bundle inicial en menos de 3 segundos
en una conexión de 10 Mbps.

**RNF-002.4** — Los endpoints de la API que consultan la BD deben usar los índices
definidos en los scripts Flyway (`email` indexed en `users`).

# RNF-003 — Mantenibilidad

**Tipo**: Requisito No Funcional de Mantenibilidad

## Requisitos Específicos

**RNF-003.1** — La cobertura de tests debe ser mínimo 80% en clases de service
y controller (medida por JaCoCo con `./mvnw verify`).

**RNF-003.2** — El código Java debe pasar Checkstyle (Google Java Style Guide)
sin errores antes de cada commit (`./mvnw checkstyle:check`).

**RNF-003.3** — El código TypeScript debe pasar ESLint sin errores antes de
cada commit (`pnpm lint`).

**RNF-003.4** — Todos los tipos TypeScript deben ser explícitos — no usar `any`
sin justificación documentada en un comentario.

**RNF-003.5** — Cada archivo de código fuente debe tener una cabecera con
¿Qué? ¿Para qué? ¿Impacto?

**RNF-003.6** — Cada método público de service y controller debe tener Javadoc
que responda ¿Qué? ¿Para qué? ¿Impacto?

**RNF-003.7** — Los scripts de migración Flyway son inmutables una vez aplicados
— nunca modificar archivos `V{n}__*.sql` ya ejecutados.

**RNF-003.8** — Las versiones de dependencias deben ser exactas (sin `^`, `~`, `*`)
en `pom.xml` y `package.json`.

# RNF-004 — Usabilidad y Accesibilidad

**Tipo**: Requisito No Funcional de Usabilidad
**Referencia**: WCAG 2.1 AA — ver `_docs/conceptos/accesibilidad-aria-wcag.md`

## Requisitos Específicos

**RNF-004.1** — La interfaz debe soportar dark mode y light mode con toggle visible
en la barra de navegación (usando la clase `dark` en `<html>` con TailwindCSS v4).

**RNF-004.2** — El contraste de color debe cumplir WCAG 2.1 AA (mínimo 4.5:1
para texto normal, 3:1 para texto grande).

**RNF-004.3** — Todos los formularios deben ser operables completamente con teclado
(Tab, Enter, Escape).

**RNF-004.4** — Los mensajes de error deben anunciarse con `role="alert"` para
lectores de pantalla.

**RNF-004.5** — El diseño debe ser responsive — funcionar correctamente en
viewport de 320 px (móvil) a 1920 px (escritorio).

**RNF-004.6** — Todos los inputs deben tener `<label>` asociado o `aria-label`
cuando el label no es visible.

**RNF-004.7** — Los botones con solo ícono deben tener `aria-label` descriptivo.

**RNF-004.8** — El acento visual del proyecto es el color **amber** (`#d97706`).
Los componentes siempre usan el token `accent-*` — nunca `amber-*` directamente.
Ver `_docs/referencia-tecnica/design-system.md`.
